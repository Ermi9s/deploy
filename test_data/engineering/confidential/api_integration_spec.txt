PURPOSE
-------
This specification defines the internal REST + WebSocket API contracts used
between microservices.  Third-party integrators must use the public API Gateway
documentation instead.


1. AUTHENTICATION
-----------------
All inter-service calls use a shared internal service token (header:
  X-Service-Token: <token>

User-facing endpoints require a JWT issued by the management service:
  Authorization: Bearer <access_token>

JWT Claims (relevant fields):
  {
    "user_id"       : 42,
    "email"         : "alice.chen@apexsolutions.com",
    "department_id" : "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "level_ranking" : 3,
    "is_superuser"  : false
  }

Token TTL: 3600 seconds (access), 7 days (refresh).


2. INGESTION SERVICE (PORT 8001)
---------------------------------
Endpoint                              Method  Description
───────────────────────────────────────────────────────────────────────
/api/documents/upload/                POST    Upload a new document
/api/documents/<id>/status/           GET     Poll ingestion status
/api/documents/<id>/register/         POST    Register file in drive
WS /ws/documents/                     WS      Real-time progress stream

Upload Request (multipart/form-data):
  file           : <binary>
  departmentAccess: {"<dept-uuid>": <min_ranking>, ...}

Status Response:
  {
    "id"       : "<uuid>",
    "status"   : "queued|processing|completed|failed",
    "progress" : 75,
    "stage"    : "embedding"
  }

5. ERROR CODES
--------------
  400 Bad Request       — Validation failure (body contains `detail`)
  401 Unauthorized      — Missing or expired JWT
  403 Forbidden         — Insufficient MAC ranking for resource
  404 Not Found         — Resource does not exist (or MAC-hidden)
  409 Conflict          — Constraint violation (e.g. user still assigned)
  429 Too Many Requests — Rate-limit exceeded (Retry-After header set)
