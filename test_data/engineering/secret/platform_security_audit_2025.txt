EXECUTIVE SUMMARY (FOR INTERNAL USE ONLY)
------------------------------------------
This report contains the findings of the annual penetration test and
security audit conducted by CyberShield Ltd. between 12–26 April 2025.
Five vulnerabilities were identified: two HIGH, two MEDIUM, one LOW.
All HIGH findings have been remediated as of the report date.

DO NOT SHARE outside the Engineering Secret clearance level.


SCOPE OF ASSESSMENT
--------------------
  • OKM Platform (all microservices, API Gateway, Elasticsearch)
  • AWS GKE production cluster (read-only access granted to auditors)
  • CI/CD pipeline (GitHub Actions configuration)
  • Network segmentation and firewall rules

DETAILED FINDINGS
------------------

F-001 — JWT Secret in CI Environment Variable (REMEDIATED)
  Description: The DJANGO_SECRET_KEY was stored as a plain-text GitHub
  Actions secret and printed in build logs due to a verbose logging flag.
  Impact: An attacker with read access to GitHub logs could forge JWTs.
  Remediation: Secret rotated; logging verbosity reduced; migrated to
  Google Secret Manager with runtime injection.

F-002 — Elasticsearch Unauthenticated Port (REMEDIATED)
  Description: Port 9200 on the GKE node pool was reachable from the
  internet due to a misconfigured Network Policy.
  Impact: Unauthenticated read/write access to all indexed document chunks.
  Remediation: Network Policy corrected; xpack.security enabled; service
  type changed to ClusterIP.

F-003 — Missing Login Rate Limit (IN PROGRESS)
  Description: The /auth/login/ endpoint accepts unlimited POST requests,
  enabling credential stuffing attacks.
  Target Fix: Django-ratelimit middleware (150 req/hour per IP) to be
  deployed in Week 3 of Q2 sprint.

F-004 — Outdated Tesseract (PLANNED Q3)
  Description: Tesseract 4.1.1 has known CVEs (CVE-2024-XXXXX).
  Target Fix: Base Docker image updated to Tesseract 5.3 in Q3.

F-005 — CORS Wildcard in Staging (REMEDIATED)
  Description: CORS_ALLOW_ALL_ORIGINS = True was set in the staging
  environment, allowing cross-origin requests from any domain.
  Remediation: CORS_ALLOWED_ORIGINS restricted to approved staging domains.


AUDITOR RECOMMENDATION
-----------------------
Implement a Security Champions program within the Engineering team.
Schedule quarterly internal security reviews between external audits.
